## 🔹 1️⃣ Bag of Words (BOW)

###  الفكرة
تقنية إحصائية بسيطة تقوم بتمثيل النص بناءً على **عدد تكرار الكلمات** دون الاهتمام بالترتيب أو المعنى.

### آلية العمل
- تقسيم النص إلى كلمات
- بناء قاموس (Vocabulary)
- إنشاء متجه يمثل عدد مرات ظهور كل كلمة

### ✅ المميزات
- سهلة وسريعة
- مناسبة كنقطة بداية

### ❌ العيوب
- لا تفهم المعنى
- لا تأخذ السياق أو الترتيب

###  الملف
```
BOW.py
```

---

## 🔹 2️⃣ TF-IDF (Term Frequency – Inverse Document Frequency)

### الفكرة
تحسين BOW عبر إعطاء وزن أعلى للكلمات المهمة ووزن أقل للكلمات الشائعة جدًا.

### ⚙️ آلية العمل
- TF: عدد مرات ظهور الكلمة في الوثيقة
- IDF: مدى ندرة الكلمة في جميع الوثائق

### ✅ المميزات
- أدق من BOW
- شائع في تصنيف النصوص واسترجاع المعلومات

### ❌ العيوب
- لا يفهم المعنى الدلالي
- يعتمد على الإحصاء فقط

### الملف
```
TF_IDF.py
```

---

## 🔹 3️⃣ Word2Vec

### 📌 الفكرة
تمثيل الكلمات كمتجهات رقمية بحيث تكون الكلمات المتشابهة في المعنى متقاربة في الفضاء المتجهي.

### آلية العمل
- Skip-Gram أو CBOW
- يعتمد على سياق الكلمة

### ✅ المميزات
- يفهم العلاقات الدلالية
- يقلل الأبعاد

### ❌ العيوب
- يحتاج بيانات كبيرة
- غير مناسب للنصوص القصيرة

> تم تطبيق Word2Vec على سورة الفاتحة لأغراض تعليمية فقط، والنتائج الدلالية محدودة بسبب صغر حجم البيانات.

###  الملف
```
Word2Vec.py
```

---

## 🔹 4️⃣ BERT (Bidirectional Encoder Representations from Transformers)

### الفكرة
نموذج عميق يعتمد على **Transformers** ويفهم السياق الكامل للكلمة من اليمين واليسار.

### آلية العمل
- تم استخدام نموذج BERT عربي مدرّب مسبقًا
- استخراج Embeddings للآيات
- حساب التشابه الدلالي بينها

### المميزات
- يفهم السياق بدقة عالية
- مناسب للنصوص القصيرة
- أفضل أداء في NLP الحديث

###  العيوب
- ثقيل حسابيًا
- يحتاج موارد أكبر

### الملف
```
BERT.py
```




## ملاحظات أكاديمية

- Word2Vec غير مناسب للتدريب على نصوص قصيرة
- BERT يعطي نتائج أكثر منطقية دلاليًا مع النصوص الصغيرة
- هذا المشروع يهدف إلى **التعلّم والمقارنة** وليس بناء نموذج إنتاجي
